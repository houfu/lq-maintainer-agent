# ci/canon-pin.txt — the CI canon pin (design doc §3.4, §4.3).
#
# Format: lines starting with '#' are comments; exactly ONE non-comment
# line, containing the full 40-character commit SHA of legalquants/lq-ai
# that this repo's CI resolves canon citations against (the drift check
# in ci/canon-drift-check.yml and, once the M1 harness lands, the eval
# fixtures in ci/eval-run.yml).
#
# This pin exists so agent-repo CI is deterministic: it must not break
# because lq-ai merged something an hour ago. It is advanced by the
# scheduled job in ci/canon-drift-check.yml, which opens a PR — never
# edit it silently; canon updates to the agent's world-model are
# reviewed like any other change. Runtime triage does NOT use this pin:
# the runtime canon is always the maintainer's own lq-ai clone at its
# main HEAD (§3.4).
